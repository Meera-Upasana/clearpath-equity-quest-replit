Set up a full-stack monorepo:

SERVER: EXPRESS API PROXY

Create /server/src/index.ts — Express server with the following route modules:

1. /api/cdc-places
GET /api/cdc-places/states
- Fetches from: https://chronicdata.cdc.gov/resource/swc5-untb.json
- Query params to forward: $where, $select, $limit (default 5000)
- Returns: Array of {locationabbr, locationdesc, category, measure, data_value, year}
- Cache: In-memory cache with 24-hour TTL using node-cache
- Error handling: On API failure, fall back to hardcoded static data file /server/src/fallback/cdc-static.ts

GET /api/cdc-places/zip/:zipcode
- Fetches county-level CDC data for a given ZIP's county FIPS
- Uses ZIP→county crosswalk table (include hardcoded top 500 US ZIP codes)

2. /api/hrsa
GET /api/hrsa/fqhcs/:zipcode
- Fetches from: https://data.hrsa.gov/api/download?fileNm=FQHC_SITES&fileType=json OR https://findahealthcenter.hrsa.gov/api/
- Returns: Array of {site_name, address, city, state, zip, lat, lng, health_center_type}
- Filter to within 10-mile radius of input ZIP using Haversine formula
- Include ZIP centroid lookup table for top 1000 US ZIPs

GET /api/hrsa/coverage-summary/:zipcode
- Returns: {zip, fqhc_count, nearest_fqhc_miles, coverage_tier: 'desert'|'limited'|'adequate'}

3. /api/usaspending
GET /api/usaspending/grants/:zipcode
- Fetches from: https://api.usaspending.gov/api/v2/search/spending_by_geography/
- POST body: {scope: "place_of_performance", geo_layer: "zip", filters: {keywords: ["healthcare","mobile clinic","diabetes","chronic disease"], time_period: [{start_date: "2022-01-01", end_date: "2024-12-31"}], place_of_performance_locations: [{country: "USA", zip: zipcode}]}}
- Returns: {zip, total_awards, total_amount, top_recipients: [{name, amount, award_date}]}
- Cache: 6-hour TTL

POST /api/usaspending/compare
- Body: {zipcodes: string[]}
- Runs the above query for each ZIP concurrently (Promise.all)
- Returns ranked comparison array sorted by total_amount desc

4. /api/competitive-intelligence
POST /api/competitive-intelligence/analyze
- Body: {target_zip: string, comparison_zips: string[]}
- Orchestrates: calls HRSA + USAspending in parallel for each ZIP
- Synthesizes result object: {winner_zip, reasoning_factors: [], fqhc_gaps: [], funding_gaps: [], recommendation_tier: 'priority_1'|'priority_2'|'monitor'}
- This is the core MCP orchestration endpoint

5. /api/grant-proposal
POST /api/grant-proposal/generate
- Body: {hospital_name, target_zip, grant_program, amount, duration_months}
- Calls /api/competitive-intelligence/analyze internally first
- Uses Anthropic Claude API (claude-sonnet-4-20250514) to generate the full grant proposal text
- System prompt: "You are a federal grant writer specializing in HRSA and CDC community health grants. Generate a complete, professional grant proposal. Use ONLY the data provided in the user message. Format as structured sections with headers. Be specific with numbers and cite the data sources."
- User message: Inject all live data fetched from CDC PLACES, HRSA, USAspending into the prompt as a structured JSON context block
- Returns: {executive_summary, needs_assessment, patient_impact, roi_model, methodology} as separate string fields
- Stream the response using SSE (server-sent events) so the frontend can show typing animation

6. /api/health
GET /api/health
- Returns status of all 4 upstream APIs by making lightweight test calls
- Returns: {cdc: 'ok'|'error', hrsa: 'ok'|'error', usaspending: 'ok'|'error', anthropic: 'ok'|'error', timestamp}

---

ERROR HANDLING PHILOSOPHY

Every API call must degrade gracefully:
- CDC PLACES offline → use /server/src/fallback/cdc-static.ts (the full 51-state dataset hardcoded)
- HRSA offline → return {fqhc_count: null, coverage_tier: 'unknown'}, show "Live data unavailable" in the card
- USAspending offline → return {total_amount: null}, show "Funding data unavailable"
- Anthropic offline → return a pre-written template proposal with [PLACEHOLDER] tokens for the admin to fill in manually

---

CORS & SECURITY

- Express: cors({origin: process.env.CLIENT_URL || 'http://localhost:5173'})
- Rate limit all /api/* routes: 100 requests per 15 minutes per IP using express-rate-limit
- Sanitize all ZIP code inputs: must match /^\d{5}$/ regex before forwarding to upstream APIs
- No API keys ever sent to the client — all upstream calls happen server-side only
